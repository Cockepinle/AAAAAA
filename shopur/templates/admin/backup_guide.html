📋 Регламент резервного копирования и восстановления

Здесь вы можете создавать бэкапы базы данных и восстанавливать данные из сохранённых дампов.

🎯 Правильный процесс:

1️⃣ ДО внесения изменений:
Нажмите кнопку "Сделать бэкап" чтобы создать копию текущего состояния БД.
2️⃣ Работаем с приложением:
Вносим изменения, добавляем товары, заказы и т.д.
3️⃣ Если ЧТО-ТО СЛОМАЛОСЬ:
Нажмите "Восстановить" и выберите бэкап из списка выше.

✅ Создать бэкап

Снимите резервную копию всех данных БД. Файл будет сохранён в папку backups/ с меткой времени.

💻 Альтернативный способ: PowerShell скрипты

В папке scripts/ находятся скрипты для резервного копирования через PostgreSQL:

📝 Создать бэкап через PowerShell:
cd scripts ; powershell -ExecutionPolicy Bypass -File backup.ps1
Создаст файл: backup_shopurpro_YYYYMMDD_HHMM.dump
🔄 Восстановить через PowerShell:
cd scripts ; powershell -ExecutionPolicy Bypass -File restore.ps1 -FilePath backup_shopurpro_20251122_175734.dump
Восстановит полную БД из нативного PostgreSQL дампа
✅ После восстановления выполнить в проекте:
python manage.py migrate
python manage.py fix_trigger
python manage.py runserver
